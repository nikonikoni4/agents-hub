# 规则文档审查清单

## 结构审查

- [ ] **行数限制**：≤ 200 行（硬性指标）
- [ ] **元数据**：是否有 `updated_at` 和 `trigger`？
- [ ] **上级链接**：子模块是否有指向上级的链接？（格式：`> 上级规则：[../CLAUDE.md]`）

## 内容审查

### 教学性内容检测（必须删除）

- [ ] **概念解释**：是否包含"什么是X"、"X的定义"等解释性内容？
- [ ] **技术教学**：是否包含"X的工作原理"、"为什么使用X"等教学内容？
- [ ] **历史背景**：是否包含"我们选择X的原因"、"过去我们使用Y"等历史信息？

### 约束检查（必须包含）

- [ ] **明确的约束**：是否有 ✅ 允许 / ❌ 禁止 的明确标注？
- [ ] **代码示例**：是否有正确 vs 错误的代码对比？
- [ ] **决策表**：是否有"何时用A / 何时用B"的决策表？
- [ ] **触发场景**：是否说明什么时候需要遵守这条规则？

## 重复内容审查

### 与上级 CLAUDE.md 重复

检查以下内容是否在上级已定义：
- [ ] 架构约束（模块隔离、单向依赖、职责分离）
- [ ] 全局禁止项
- [ ] 决策表
- [ ] 目录结构说明

### 与 docs/ARCHITECTURE.md 重复

- [ ] 是否重复说明架构设计？
- [ ] 是否重复说明模块职责？

## 架构一致性审查

- [ ] **符合架构约束**：规则是否与 ARCHITECTURE.md 的架构约束一致？
- [ ] **符合现有代码风格**：规则是否与现有代码风格一致？
- [ ] **无冲突**：规则是否与其他规则冲突？

## 审查报告模板

```markdown
## 审查报告：<文件路径>

### ✅ 符合要求
- <列出符合要求的部分>

### ❌ 需要修改
1. <问题描述>（第 X-Y 行）
   - 建议：<修改建议>

### 📊 统计
- 当前行数：X 行
- 建议行数：≤ 200 行
- 需要删除：约 Y 行
```
