# Hand-on Skill

从其他 agent 接手任务，读取 hand-off 文档快速恢复上下文并继续执行。

## 使用场景

- 用户："接手这个任务" + 提供 hand-off 文档路径
- 用户："继续上次的工作"
- 用户："从上次中断的地方继续"

---

## 执行流程

### 1. 定位 hand-off 文档

**优先级**：
1. 用户直接提供的路径
2. `docs/temp/hand-off/` 目录下最新的文件
3. 如果有多个文件，列出让用户选择

```
ls -lt docs/temp/hand-off/ | head -5
```

### 2. 解析文档结构

读取 hand-off 文档，提取以下区域：

| 区域 | 用途 |
|------|------|
| **元数据** | 时间、分支、状态 → 恢复工作环境 |
| **任务目标** | 理解核心目标 |
| **已完成的工作** | 标记为已完成的 todolist 项 |
| **阻塞问题** | 优先处理的障碍 |
| **未完成的任务** | 创建 todolist 的依据 |
| **下一步行动** | 确定执行顺序 |
| **相关文件** | 快速定位代码 |
| **决策记录** | 理解之前的决策 |
| **注意事项** | 避免踩坑 |

### 3. 输出上下文摘要

向用户展示接手摘要：

```markdown
## 接手摘要

**任务**：<任务标题>
**状态**：<进行中/阻塞>
**分支**：<分支名>

### 已完成
- ✅ <已完成项 1>
- ✅ <已完成项 2>

### 待完成
- [ ] <待办项 1>
- [ ] <待办项 2>

### 阻塞（如有）
- ❌ <阻塞问题描述>

### 下一步
1. <最优先行动>
2. <次优先行动>

---
是否继续？
```

### 4. 创建任务清单

使用 TaskCreate 将未完成的任务转化为 todolist：

```python
# 对于每个未完成的任务
TaskCreate(
    subject="<任务名称>",
    description="<具体要求>"
)

# 如果有阻塞问题，创建优先任务
TaskCreate(
    subject="解决阻塞：<问题标题>",
    description="<问题详情和相关文件>"
)
```

**任务排序**：
1. 阻塞问题（最高优先级）
2. 优先级 1 的任务
3. 优先级 2 的任务
4. 其他任务

### 5. 环境检查

根据元数据检查当前环境：

- [ ] 分支是否正确？（如果不正确，提示切换）
- [ ] 相关文件是否存在？
- [ ] 依赖是否安装？

### 6. 开始执行

根据 todolist 开始执行任务，从最高优先级开始。

---

## 输出格式

### 接手成功

```markdown
✅ 已接手任务：<任务标题>

📋 任务清单已创建（N 项）
📁 相关文件已定位（N 个）

当前分支：<branch>
下一步：<第一个待办任务>

开始执行...
```

### 接手失败

```markdown
❌ 无法接手任务

原因：<具体原因>

建议：
1. <建议的解决方案>
```

**常见失败原因**：
- hand-off 文档不存在或格式错误
- 分支不存在
- 关键文件缺失

---

## 重要约束

1. **先确认再执行**：展示摘要后等待用户确认
2. **保持连续性**：继承 hand-off 中的决策，不重新讨论已决定的事项
3. **优先解决阻塞**：如果有阻塞问题，优先处理
4. **不重复已完成**：已完成的任务直接跳过
5. **记录进展**：执行过程中更新 todolist 状态

---

## 与 hand-off 的关系

hand-on 解析 hand-off 生成的文档。文档必须遵循 hand-off 定义的标准格式：

```
hand-off (输出) → 标准格式文档 → hand-on (输入)
```

如果文档格式不符合标准，提示用户先运行 hand-off 重新生成。
