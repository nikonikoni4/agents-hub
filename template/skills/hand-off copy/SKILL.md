# Hand-off Skill

将当前 session 的工作内容总结为标准格式的交接文档，供其他 agent 接手。

## 输出规范

- **路径**：`docs/temp/hand-off/`
- **命名**：`YYYY-MM-DD-<topic>.md`（用户指定 topic 或自动生成）
- **格式**：必须遵循下方的「标准格式模板」

---

## 标准格式模板

```markdown
# <任务标题>

**时间**：YYYY-MM-DD
**分支**：<当前分支名>
**状态**：✅ 已完成 / 🚧 进行中 / ❌ 阻塞

---

## 一、任务目标

<一句话描述本次任务的核心目标>

---

## 二、已完成的工作

### <模块/阶段名称>
- [x] <完成项 1>
- [x] <完成项 2>

### <模块/阶段名称>
- [x] <完成项 3>

---

## 三、阻塞问题（如有）

### 问题 N：<问题标题>

**现象**：<具体表现>
**调用链**：<相关调用路径>
**根本原因**：<分析出的原因>
**相关文件**：
| 文件 | 说明 |
|------|------|
| `path/to/file` | 文件作用 |

---

## 四、未完成的任务

### 优先级 1：<任务名称>

- [ ] <待办项 3>

---

## 五、下一步行动

1. <最优先的行动>

3. <其他行动>

---

## 六、相关文件
```

## 执行流程

### 1. 收集信息

- 读取当前 session 的对话历史
- 确定任务主题（用户指定或自动提取）
- 获取当前分支名、日期

### 2. 提取内容

按照标准格式提取：

| 区域 | 来源 |
|------|------|
| 任务目标 | 用户最初的请求 |
| 已完成的工作 | AI 执行的操作、创建/修改的文件 |
| 阻塞问题 | 未解决的错误、报错信息 |
| 未完成的任务 | 用户提到但未完成的需求 |
| 下一步行动 | 建议的后续步骤 |
| 相关文件 | 本次修改的所有文件 |
| 决策记录 | 讨论过的重要决策 |
| 注意事项 | 需要特别注意的点 |

### 3. 格式校验

**必须包含**：
- ✅ 元数据（时间、分支、状态）
- ✅ 已完成的工作
- ✅ 未完成的任务
- ✅ 下一步行动
- ✅ 相关文件

**可选包含**：
- 阻塞问题（有阻塞时必须写）
- 决策记录（有重要决策时必须写）
- 注意事项

### 4. 输出确认

向用户展示交接文档摘要，确认：
1. 任务主题是否准确
2. 已完成/未完成的划分是否正确
3. 是否有遗漏的重要信息

### 5. 写入文件

确认后写入 `docs/temp/hand-off/YYYY-MM-DD-<topic>.md`

---

## 重要约束

1. **只总结，不评判**：客观记录工作内容，不添加主观评价
2. **保留上下文**：关键的背景信息要保留，接手 agent 不需要重新探索
3. **任务可执行**：未完成的任务要具体到可以直接执行
4. **文件路径准确**：所有文件路径必须是相对于项目根目录的相对路径
5. **阻塞问题详细**：如果存在阻塞，必须提供足够的调试信息

---

## 与 hand-on 的关系

hand-off 生成的文档是 hand-on 的输入。hand-on 会解析此文档：
- 元数据 → 恢复上下文
- 已完成/未完成 → 创建 todolist
- 阻塞问题 → 优先处理
- 相关文件 → 快速定位代码
