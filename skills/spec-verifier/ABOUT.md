# Spec Verifier Skill 介绍

## 概述

Spec Verifier 是一个 Claude Code skill，用于验证 AI 生成的文档是否符合用户的原始需求和意图。它通过分析会话历史，自动检测文档对用户需求的覆盖情况，帮助用户确保 AI 正确理解了他们的需求。

## 核心功能

### 1. 会话历史分析

- 读取 Claude Code 会话文件
- 提取用户的原始需求
- 识别 AI 生成的文档

### 2. 需求覆盖检测

- 智能匹配用户需求与 AI 文档
- 计算需求覆盖率
- 识别遗漏的需求点

### 3. 验证报告生成

- 详细的验证结果
- 符合度评分
- 改进建议

## 使用场景

### 场景 1：验证 Spec 文档

```
用户: 我需要一个用户登录功能
AI: [生成 spec 文档]
用户: 验证一下刚才写的 spec
AI: [启动 spec-verifier，生成验证报告]
```

### 场景 2：检查计划文档

```
用户: 制定一个项目计划
AI: [生成计划文档]
用户: 检查文档是否符合我的想法
AI: [启动 spec-verifier，验证计划]
```

### 场景 3：审查技术方案

```
用户: 设计一个微服务架构
AI: [生成技术方案]
用户: 审查计划文档

```

   - 改进建议
   - 评分结果

## 最佳实践

### 1. 明确需求

用户需求越明确，验证结果越准确。

### 2. 完整文档

AI 文档越完整，覆盖率越高。

### 3. 定期验证

建议在文档生成后立即验证。

### 4. 反馈改进

根据验证结果改进文档。

## 故障排除

### 常见问题

1. **会话文件找不到**
   - 检查 session_id
   - 确认 CLAUDE_CONFIG_DIR

2. **匹配率过低**
   - 检查需求明确性
   - 调整匹配阈值

3. **脚本错误**
   - 检查 Python 环境
   - 确认依赖包
