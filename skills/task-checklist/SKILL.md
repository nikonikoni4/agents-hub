# Task Checklist Skill

任务检查清单的创建与验证。

## 文档路径规范

```
docs/task-check-list/YYYY-MM-DD-<task-slug>.md
```

- `YYYY-MM-DD`：任务日期
- `<task-slug>`：任务简短描述，用连字符分隔（如 `fix-login-bug`）

## 操作一：创建检查清单

### 触发条件
用户要求创建 checklist、总结任务关键点、或会话中存在明确的多步骤任务。

### 步骤

1. **回顾会话内容**：提取当前会话中与任务相关的所有关键点
2. **确定任务信息**：
   - 任务背景：为什么要做这个任务
   - 任务目标：最终要达成什么
3. **生成清单项**：每项必须是可验证的具体动作，不要包含无关内容
4. **写入文件**：路径为 `docs/task-check-list/YYYY-MM-DD-<task-slug>.md`

### 文档模板

```markdown
# <任务标题>

## 任务背景

<为什么要做这个任务，来源是什么>

## 任务目标

<最终要达成的效果>

## 检查清单

- [ ] 具体检查项 1
- [ ] 具体检查项 2
- [ ] 具体检查项 3
```

### 清单项编写原则

- 每项必须是**可验证的**：有明确的完成标准
- 每项必须是**具体的**：不含糊，不笼统
- 只包含**当前任务**的内容，不混入无关事项

## 操作二：验证检查清单

### 触发条件
用户要求验证 checklist、检查任务完成情况、review 任务。

### 步骤

1. **定位文档**：
   - 用户指定了文件 → 读取该文件
   - 未指定 → 读取 `docs/task-check-list/` 下最新的文件（按文件名日期排序）
2. **逐项验证**：对每个 `- [ ]` 项，通过以下方式确认完成情况：
   - 检查相关代码文件是否存在/修改
   - 检查功能是否实现
   - 检查测试是否通过
3. **更新状态**：将验证通过的项标记为 `- [x]`
4. **输出报告**：汇总验证结果

### 验证报告格式

```
## 验证结果

✅ 已完成：X 项
❌ 未完成：Y 项
⚠️ 部分完成：Z 项

### 详情
- [x] 检查项 1 — 已在 xxx 文件中实现
- [ ] 检查项 2 — 未找到相关实现
```
