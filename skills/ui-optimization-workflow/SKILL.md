# UI 优化工作流程

## 概述

本文档描述 Manager 协调团队进行前端 UI 优化的标准流程。涉及三个角色：

- **Manager**：统筹协调，分配任务，闭环验收
- **UI界面设计审核**：截图分析，对比设计规范，提出修改意见
- **前端代码编写执行者**：执行 CSS/组件修改
- **前端界面UX和元素审核者**：验证元素完整性和交互未被破坏

---

## 标准流程（每轮）

```
┌─────────────────────────────────────────────────────────────┐
│  第1步：UI审核截图分析                                        │
│  ├─ 生成截图（python scripts\screenshot_main.py）            │
│  ├─ 用 mimo-image MCP 读取截图和参考图                        │
│  └─ 输出：问题清单 + 改进建议 + 评分                          │
├─────────────────────────────────────────────────────────────┤
│  第2步：前端执行修改                                          │
│  ├─ 根据 UI 审核的问题清单修改 CSS/组件                       │
│  └─ 输出：修改文件列表 + 改动说明                             │
├─────────────────────────────────────────────────────────────┤
│  第3步：UX元素审核                                            │
│  ├─ 验证原有元素是否完整保留                                  │
│  ├─ 验证交互是否未被破坏                                      │
│  └─ 输出：通过/不通过 + 问题列表                              │
└─────────────────────────────────────────────────────────────┘
        ↓ 重复 4-6 轮，直到评分达标 ↓
```

## 关键注意事项

### 1. 必须使用 Mimo-Image MCP 看图片

**绝对不能直接读取图片文件**（如 `Read` 工具读 PNG），必须通过 mimo-image MCP 的 `understand_image` 工具。

**正确用法：**
```
mcp__mimo-image__understand_image(
  image_path="D:\\path\\to\\screenshot.png",
  prompt="描述这张图片的UI设计特征"
)
```

**错误用法：**
```
Read(file_path="D:\\path\\to\\screenshot.png")  // ❌ 禁止
```

**为什么：**
- 直接读取图片文件会返回二进制数据，无法分析
- mimo-image MCP 会将图片转为文字描述，供 AI 分析和对比

### 2. 如何使用截图

#### 生成截图

运行 Python 脚本生成当前界面截图：

```bash
python scripts\screenshot_main.py
```

截图保存位置：`screenshots/main.png`

#### 截图文件说明

| 文件 | 用途 |
|------|------|
| `screenshots/main.png` | 当前主界面截图（每次运行脚本会覆盖） |
| `screenshots/风格参考图.png` | 设计风格参考图（固定不变） |
| `screenshots/role_management_teams.png` | 角色管理团队页面截图 |

#### 对比审核流程

1. **先运行截图脚本**：`python scripts\screenshot_main.py`
2. **用 mimo-image 读取当前截图**：分析当前 UI 问题
3. **用 mimo-image 读取参考图**：了解目标风格
4. **对比两者差异**：给出具体问题清单和改进建议

### 3. UI 审核要求

- **严格对比**：不能只说"看起来不错"，必须逐项对比颜色、圆角、阴影、间距
- **量化指标**：给出具体数值（如"圆角应从 8px 改为 12px"）
- **优先级排序**：P0（必须修）、P1（应该修）、P2（可以修）
- **评分系统**：1-10 分，每轮对比上一轮的提升

### 4. 前端执行要求

- **只改 CSS，不改逻辑**：UI 优化阶段不动组件业务逻辑
- **使用 CSS 变量**：不硬编码颜色，使用 `var(--xxx)`
- **双主题适配**：浅色和深色主题都要考虑
- **最小改动**：只改 UI 审核指出的问题，不做额外修改

### 5. UX 审核要求

- **元素完整性**：所有原有元素必须保留，不能删除
- **交互完整性**：所有原有交互必须正常工作
- **样式变更影响**：检查 CSS 修改是否意外影响了其他组件

## Manager 派发任务模板

### 派发给 UI 审核

```
UI界面设计审核，第N轮审核任务：

## 步骤1：截图
运行 python scripts\screenshot_main.py 生成截图。

## 步骤2：对比审核
用 mimo-image MCP 读取：
- 当前截图：screenshots/main.png
- 风格参考图：screenshots/风格参考图.png

## 本轮改进内容
[列出本轮做了哪些改动]

## 审核要求
[列出需要重点审核的方面]

请给出：改进效果评价、仍需改进的问题、整体评分（1-10）
```

### 派发给前端执行者

```
前端执行者，第N轮修改任务：

## 需要修改的文件
[列出具体文件和改动]

## 改动说明
[说明每处改动的理由]

请逐一修改并确认。
```

### 派发给 UX 审核

```
前端界面UX和元素审核者，请执行UX审核：

## 审核范围
[说明审核哪个页面/组件]

## 需要验证的内容
1. 元素完整性
2. 交互验证
3. 样式变更影响检查

## 本轮CSS修改
[列出本轮做了哪些CSS改动]

给出审核结论：通过/不通过。
```

---

## 评分标准

| 分数 | 等级 | 说明 |
|------|------|------|
| 9-10 | 优秀 | 完美匹配设计规范，无明显问题 |
| 8-8.5 | 良好 | 基本匹配，有少量微调空间 |
| 7-7.5 | 合格 | 大方向正确，有明确改进点 |
| 6-6.5 | 需改进 | 有明显问题，需要多轮优化 |
| 5以下 | 不合格 | 需要重新审视设计方向 |

## 实际案例

### 主界面 UI 优化（5轮）

| 轮次 | 评分 | 主要改动 |
|------|------|---------|
| 初始 | 6/10 | 基础实现 |
| 第1轮 | 6.5/10 | 添加 --bg-hover、圆角、阴影 |
| 第2轮 | 7/10 | 增加呼吸感、输入框改 textarea |
| 第3轮 | 7.5/10 | 阴影增强、圆角微调 |
| 第4轮 | 7.5/10 | 输入框重新设计（浅灰背景、无边框） |
| 最终 | 8.5/10 | 输入框圆角 8px、整体协调 |

### 颜色/选中态调整

- 强调色：`#F57C00`（橙色） → `#8BA5BF`（柔和蓝灰）
- 选中态：蓝色块 → `rgb(232,232,232)`（灰色）
- 左侧栏：添加右侧圆角 `12px`
