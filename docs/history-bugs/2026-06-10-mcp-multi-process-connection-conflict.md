# MCP 多进程访问导致连接冲突和消息卡住

## 更新时间
2026-06-10

## 触发规则
多个进程同时访问同一个 MCP 服务器，其中一个进程突然中断连接时

## 问题描述
当多个进程（如两个 Claude CLI 实例）同时连接同一个 MCP 服务器时，如果其中一个进程突然关闭或中断，会导致：
1. 消息发送成功但 Agent 不响应
2. 消息排队但不处理，延迟数分钟
3. 出现 ConnectionResetError: [WinError 10054] 远程主机强迫关闭了一个现有的连接
4. 出现 OSError: [WinError 64] 指定的网络名不再可用

## 复现步骤
1. 启动第一个 Claude CLI 进程连接 MCP 服务器
2. 启动第二个 Claude CLI 进程连接同一个 MCP 服务器
3. 在群聊中发送消息
4. 突然关闭其中一个进程
5. 观察另一个进程的消息处理情况

## 日志表现
```
2026-06-10 11:26:55 - 创建调用 69075459: user -> manager, 类型=task
2026-06-10 11:27:53 - ConnectionResetError: [WinError 10054] 远程主机强迫关闭了一个现有的连接
2026-06-10 11:29:35 - Agent manager 开始处理消息（延迟约 3 分钟）
```

## 可能原因
1. **MCP 连接竞争**：多个进程同时访问 MCP 服务器时，连接状态不稳定
2. **连接中断未恢复**：一个进程关闭时，MCP 服务器的连接被强制断开，但另一个进程的连接未正确恢复
3. **消息队列阻塞**：消息被发送到队列，但由于连接问题，Agent 未被激活处理

## 解决方案
1. **避免多进程同时访问**：确保同一时间只有一个进程连接 MCP 服务器
2. **优雅关闭进程**：关闭进程时，先断开 MCP 连接，再关闭进程
3. **连接重连机制**：MCP 客户端应实现自动重连机制，检测连接断开后自动恢复
