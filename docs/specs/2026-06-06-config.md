## Overview

Config 模块是系统的全局配置中心，以单例模式提供统一的配置访问入口。负责管理数据路径策略、MCP 端口、Docker 沙箱参数等系统级配置，并定义跨模块共享的 Agent 平台和角色类型枚举。

## Scope

**职责边界**：

- 数据路径策略：三级路径优先级解析（migration > packaged > dev）
- 配置持久化：将用户自定义配置写入 YAML 文件
- 配置聚合：通过 `Config` 类提供统一快捷访问入口
- 枚举定义：`AgentPlatform`、`RoleType` 跨模块共享
- CLI 路径常量：宿主机与 Docker 容器内的 CLI 工具路径映射
- User 名称规范化：展示名后缀处理和身份判断

**不负责**：

- Agent 业务逻辑配置（由 roles 模块负责）
- 群聊运行时配置（由 core 层负责）
- 前端配置管理

## Core Behavior

### 三级路径优先级策略

数据存储路径 (`data_path`) 按以下优先级解析，高优先级存在时忽略低优先级：

1. **Migration 路径**：用户通过 `set_data_path()` 自定义的路径，持久化到 `config.yaml` 的 `data_path` 字段。当该字段非 `null` 时始终优先使用。
2. **Packaged 路径**：打包环境（`sys.frozen` 为 True）下的默认路径，固定为 `%LOCALAPPDATA%/AgentsHub/data/`。
3. **Dev 路径**：开发环境下的默认路径，固定为项目根目录下的 `local_data/`。

### 配置持久化机制

- 配置文件路径：开发环境为 `local_data/config/config.yaml`，打包环境为 `%LOCALAPPDATA%/AgentsHub/config/config.yaml`（配置文件本身不随数据路径迁移）
- 加载时以 YAML 文件中的值覆盖内存默认值，`null` 值不覆盖
- 保存时过滤掉 `null` 值，只持久化用户明确设置过的配置项
- 可写配置项（`data_path`、`mcp_port`、`default_user_name`、`use_docker`）在 setter 中自动触发保存

### User 名称处理规则

- 展示名后缀：除默认值 `"user"` 外，所有用户身份名自动追加后缀 `"(user)"`
- 规范化：`normalize_user_name()` 去除后缀 `"(user)"` 和首尾空格，用于身份比较
- 判断逻辑：`is_user_name()` 对给定名称和已配置名称均做规范化后比较，匹配 `"user"` 或配置值即视为 user 身份

## Technical Contract

### 配置项列表

| 字段名 | 类型 | 默认值 | 语义 |
|--------|------|--------|------|
| `data_path` | Path / None | None | 数据存储路径。None 表示使用环境默认路径 |
| `mcp_port` | int | 8765 | MCP 服务器运行端口 |
| `default_manager_name` | str | `"manager"` | 默认 manager 角色名 |
| `default_user_name` | str | `"user"` | 默认用户身份名 |
| `docker_image` | str | `"ai-tools:latest"` | Docker 沙箱镜像名称 |
| `use_docker` | bool | True | 是否默认使用 Docker 沙箱执行 |

### User 名称常量

| 常量 | 值 | 语义 |
|------|----|------|
| `USER_DISPLAY_SUFFIX` | `"(user)"` | 用户展示名的后缀标识 |

### 枚举定义

#### AgentPlatform

Agent 平台枚举，标识底层 CLI 工具类型。

| 枚举值 | 字符串值 | 语义 |
|--------|----------|------|
| `CLAUDE` | `"claude"` | Claude Code CLI |
| `CODEX` | `"codex"` | Codex CLI |
| `OPENCODE` | `"opencode"` | OpenCode CLI |

#### RoleType

角色类型枚举，标识 Agent 在团队中的角色。

| 枚举值 | 字符串值 | 语义 |
|--------|----------|------|
| `LEADER` | `"leader"` | 领导者角色，负责任务分派和协调 |
| `TEAM_MEMBER` | `"team_member"` | 团队成员角色，执行具体任务 |
| `SYSTEM` | `"system"` | 系统角色，由系统预置的特殊角色 |

### CLI 路径映射

#### 宿主机 CLI 路径

| 常量 | 路径 | 语义 |
|------|------|------|
| `CODEX_COMMAND` | `%APPDATA%/npm/codex.cmd` | 宿主机 Codex CLI 路径（npm 全局安装） |
| `CLAUDE_COMMAND` | `~/.local/bin/claude` | 宿主机 Claude CLI 路径 |
| `OPENCODE_COMMAND` | `%APPDATA%/npm/opencode.cmd` | 宿主机 OpenCode CLI 路径（npm 全局安装） |

#### Docker 容器内 CLI 路径

| 常量 | 路径 | 语义 |
|------|------|------|
| `DOCKER_CLAUDE_COMMAND` | `/usr/bin/claude` | 容器内 Claude CLI 路径（npm 全局安装） |
| `DOCKER_CODEX_COMMAND` | `/usr/bin/codex` | 容器内 Codex CLI 路径（npm 全局安装） |
