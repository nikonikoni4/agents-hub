消息流转与持久化规格

所有消息通过 GroupChat 统一包装投递并保存，MessageRouter 为纯投递层

1

核心原则

三条不可违反的设计约束
  • MessageRouter 是纯投递层 — 只负责将消息投递到目标 Agent 的队列，不承担业务逻辑
  • GroupChat 提供统一包装方法 — 所有通过 MessageRouter 投递的消息都通过 GroupChat.send_message_to_agent() 完成投递和保存
  • 所有消息都保存到群聊历史 — 确保完整的消息记录供前端展示和上下文管理

① user → agent

前端 → API → GroupChat.send_message_to_agent() 投递+保存 user 消息 → Agent 处理 → finish_agent_call 保存回复

② agent → agent

Agent A → call_agent → GroupChat.send_message_to_agent() 投递+保存 TASK → Agent B 处理 → NOTIFICATION 通过同一包装方法投递+保存

③ 群聊公开发言

speak_in_group_chat 直接调用 add_message()，不经过 MessageRouter

统一规则

所有消息都保存到群聊历史。投递由 MessageRouter 负责，保存由 GroupChat 包装层统一处理。

3

MessageRouter 职责边界

纯投递层的 do / don't
只负责
  • 验证消息格式（内容非空、发送方和接收方已注册）
  • 投递消息到目标 Agent 的队列
  • 处理投递失败（队列满、目标不存在）
不负责
  • 决定哪些消息需要保存到群聊历史
  • 调用 GroupChatContext.add_message()
  • 区分消息类型（TASK / NOTIFICATION）
  • 判断调用方是 user 还是 agent
  • 依赖 GroupChatContext 或任何业务层组件
