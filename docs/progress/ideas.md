### 💭 2026-06-08 单聊 Continue 模式的 Agent 状态约束
- **来源**：用户直接输入
- **内容**：单聊创建需要按模式区分约束：
  - **全新聊天 / Fork**：无约束
  - **Continue**：必须有约束
    1. **创建时**：后端检查 Agent 状态，若处于工作状态则拒绝请求；若空闲则创建单聊，将 Agent 状态改为"单聊中"，并停止 Agent Run
    2. **进行中**：其他 Agent 通过 callagent 派活给该 Agent 时，直接拒绝并告知"当前 Agent 正在与 User 单聊，无法执行任务"（后续可改为任务排队）
    3. **群聊中 @ 该 Agent**：前端提示"Agent 正在单聊中，请关闭单聊后再在群聊中发送消息"
    4. **结束条件**：前端主动停止单聊，或长时间无新消息自动超时停止。结束后 Agent 状态变回空闲，重新启动 Agent Run
  - **Agent 状态标志位**：复用 `is_process`，扩展为三种状态：空闲 / 工作中 / 单聊中
- **类型**：需求
- **备注**：改动涉及面较广（callagent 防护、Agent 状态机、群聊消息拦截、单聊超时机制），先记录，后续拆分实现
