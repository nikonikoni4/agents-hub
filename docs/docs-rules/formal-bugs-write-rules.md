# formal-bugs-write-rules

向 `docs/history-bugs` 写入正式 Bug 记录时，需简明扼要地包含以下结构：

1. **Bug简述**：一句话概括问题现象及影响。
2. **复用场景**：指出该修复方案可供借鉴的其他模块或场景。
3. **代码位置**：精确定位发生 Bug 的文件路径、模块或行数。
4. **发生原因**：简练分析导致 Bug 的根本原因。
5. **最佳方案**：提供清晰的代码修复方案或最终解决办法。
