# OpenCode 流式输出格式

## 测试环境

- 命令: `opencode run --format json`
- 模型: mimo-v2.5-pro
- 输出格式: JSON 事件流

## 输出格式

每个事件是一个 JSON 对象，包含以下字段：

```json
{
  "type": "事件类型",
  "timestamp": 时间戳,
  "sessionID": "会话ID",
  "part": { ... }
}
```

## 事件类型

### 1. step_start - 步骤开始

```json
{
  "type": "step_start",
  "timestamp": 1781008067675,
  "sessionID": "ses_153a5a46effe5Y0RDimE82M9HH",
  "part": {
    "id": "prt_eac5a6c59001xRnw7p5PRQri2F",
    "messageID": "msg_eac5a5c59001HPULeIkaQBvVct",
    "sessionID": "ses_153a5a46effe5Y0RDimE82M9HH",
    "snapshot": "405e4bebb1724289e535a4efe9b263176551930b",
    "type": "step-start"
  }
}
```

### 2. text - 文本内容（流式）

```json

  "timestamp": 1781008069322,
  "sessionID": "ses_153a5a46effe5Y0RDimE82M9HH",
```

## 与 Agent Bridge 集成要点

1. **进程调用**: 通过子进程调用 `opencode run --format json`
2. **流式解析**: 逐行读取 stdout，解析 JSON 事件
3. **事件处理**:
   - `step_start`: 初始化响应对象
   - `text`: 累积文本内容，可实时返回给调用方
   - `step_finish`: 完成响应，提取 token 使用信息
4. **会话管理**: `sessionID` 可用于多轮对话
5. **错误处理**: 检查进程退出码和 stderr

## Agent Bridge 集成实现

### 文件结构

```
agents_hub/agent_bridge/
├── executors/
│   └── opencode.py      # OpenCode 执行器
├── parsers/
│   └── opencode.py      # OpenCode 解析器
└── bridge.py            # 统一接口（已更新）
```

### 核心组件

#### 1. OpenCodeExecutor (`executors/opencode.py`)

- 通过子进程调用 `opencode run --format json`
- 支持 `OPENCODE_CONFIG_DIR` 环境变量设置系统提示词
- 返回解析后的事件字典流

#### 2. OpenCodeParser (`parsers/opencode.py`)

- 将 OpenCode 事件转换为统一的 `StreamEvent` 格式
- 事件映射：
  - `step_start` → `AgentEventType.INIT`
  - `text` → `AgentEventType.TEXT_DELTA`
  - `step_finish` → `AgentEventType.TURN_COMPLETE`

#### 3. AgentPlatform 枚举 (`config/types.py`)

```python
class AgentPlatform(Enum):
    CLAUDE = "claude"
    CODEX = "codex"
    OPENCODE = "opencode"  # 新增
```

### 使用示例

```python
from agents_hub.agent_bridge import AgentBridge
from agents_hub.roles.models import RoleConfig
from agents_hub.config.types import AgentPlatform

bridge = AgentBridge()

config = RoleConfig(
    name="test-agent",
    platform=AgentPlatform.OPENCODE,
    work_root="D:\\测试",  # OPENCODE_CONFIG_DIR，包含 agents/ 目录
)

# system_prompt 传入 agent 名称（对应 agents/ 目录下的 .md 文件名）
async for event in bridge.execute_stream("你好", config, system_prompt="nico"):
    if event.type == AgentEventType.TEXT_DELTA:
        print(event.content["text"], end="", flush=True)
```

**说明：**
- `config.work_root`: 设置 `OPENCODE_CONFIG_DIR` 环境变量，指向包含 `agents/` 目录的配置目录
- `system_prompt`: 传入 agent 名称（如 "nico"），对应 `agents/nico.md` 文件
- agent 文件内容即为系统提示词，可动态编写
