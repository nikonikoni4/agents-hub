核心原则

错误处理的 5 个基本原则

1. 异常是异常，不是返回值

原则：用异常表示"不应该发生的情况"，用返回值表示"正常的分支"

❌ 错误示范
def get_user(user_id: str) -> dict | None:
    if user_id not in users:
        return None  # 用户不存在是异常情况
    return users[user_id]
✅ 正确示范
def get_user(user_id: str) -> dict:
    if user_id not in users:
        raise UserNotFoundError(user_id)
    return users[user_id]
为什么？
  • 异常会强制调用者处理错误情况
  • 返回值容易被忽略（忘记检查 None）
  • 异常可以携带更多上下文信息

2. 在边界处理

原则：内部函数抛出异常，在系统边界统一捕获并转换

边界在哪里？

  • MCP Tool 入口（call_agent）
  • REST API 端点（FastAPI 路由）
  • Agent.run() 循环
  • 文件操作函数

3. 保留上下文

原则：异常应该携带足够的信息用于调试和日志记录

上下文包括什么？

  • 错误发生时的参数值
  • 相关的业务对象 ID
  • 可用的替代选项
  • 原始异常（异常链）

4. 分类处理

原则：不同类型的错误用不同的处理策略

错误类型处理策略示例
可恢复错误自动重试（带退避）API 限流、临时网络故障
业务错误返回给调用者处理Agent 不存在、权限不足
验证错误返回详细提示参数格式错误、数据不符合约束
系统错误记录日志、通知管理员数据库连接失败、磁盘满

5. 对用户友好

原则：错误信息应该清晰、可操作

❌ 信息模糊
{"error": "Invalid input"}
✅ 清晰 + 可操作
{
  "error_code": "AGENT_NOT_FOUND",
  "message": "Agent 'worker1' 不存在",
  "details": {
    "agent_name": "worker1",
    "available_agents": ["小李", "小赵", "Leader"]
  },
  "suggestion": "请使用 list_agents 查看可用列表"
}

异常层次结构设计

建立清晰的异常继承树

设计原则

核心思想：建立清晰的异常继承树，让调用者可以选择捕获的粒度

AgentsHubError (基类) ├── ValidationError (验证错误) ├── ResourceNotFoundError (资源不存在) ├── StateError (状态错误) ├── ExternalServiceError (外部服务错误) └── SystemError (系统错误)
为什么分层？
  • 粗粒度捕获：except ValidationError 捕获所有验证错误
  • 细粒度捕获：except AgentNotFoundError 捕获特定错误
  • 全局捕获：except AgentsHubError 捕获所有业务异常

基础异常类设计

class AgentsHubError(Exception):
    """所有 agents-hub 异常的基类"""

    def __init__(
        self,
        message: str,
        error_code: str | None = None,
        details: dict | None = None,
        cause: Exception | None = None
    ):
        self.message = message
        self.error_code = error_code or self.__class__.__name__
        self.details = details or {}
        self.cause = cause
        super().__init__(message)

    def __str__(self) -> str:
        base = f"[{self.error_code}] {self.message}"
        if self.details:
            base += f" | Details: {self.details}"
        if self.cause:
            base += f" | Caused by: {type(self.cause).__name__}: {self.cause}"
        return base

    def to_dict(self) -> dict:
        return {
            "error_code": self.error_code,
            "message": self.message,
            "details": self.details,
            "type": self.__class__.__name__
        }
关键设计点
  • error_code：用于 API 响应，客户端可以根据错误码做不同处理
  • details：携带上下文信息，便于调试
  • cause：保留原始异常，形成异常链
  • to_dict()：方便转换为 JSON 响应

错误分类策略

按"谁应该处理"分类

这是最实用的分类方式，直接指导错误处理策略。

1. 可恢复错误（Recoverable Error）

特征：系统可以自动重试解决

处理策略：自动重试（带指数退避）

示例：API 限流（429 错误）、临时网络故障、队列临时满

2. 业务错误（Business Error）

特征：业务逻辑错误，需要调用者处理

处理策略：返回结构化错误信息给调用者

示例：Agent 不存在、权限不足、操作超时

3. 验证错误（Validation Error）

特征：输入参数不符合要求

处理策略：返回详细的错误信息，帮助调用者修正

示例：参数缺失或格式错误、数据不符合约束、消息内容为空

4. 系统错误（System Error）

特征：系统级故障，需要人工介入

处理策略：记录日志、通知管理员、返回通用错误信息（不暴露内部细节）

示例：数据库连接失败、磁盘空间不足、配置文件损坏

错误类型特征处理策略恢复方式
可恢复临时性问题自动重试指数退避
业务错误逻辑错误返回调用者人工修正
验证错误输入问题详细提示修正输入
系统错误基础设施告警通知管理员介入

异常处理模式

5 种核心异常处理模式

模式 1：边界处理（Boundary Handling）

核心思想：在系统边界捕获所有异常，转换为统一格式

适用场景：MCP Tool 入口、REST API 端点

def call_agent(group_chat_id, send_from, send_to, content) -> dict:
    """MCP Tool 入口，必须返回结构化响应"""
    try:
        result = _do_call_agent(group_chat_id, send_from, send_to, content)
        return {"success": True, "data": result}

    except ValidationError as e:
        logger.warning(f"验证错误: {e}")
        return {"success": False, "error": e.to_dict(),
                "suggestion": "请检查输入参数是否正确"}

    except ResourceNotFoundError as e:
        logger.warning(f"资源不存在: {e}")
        return {"success": False, "error": e.to_dict(),
                "suggestion": "请使用 list_agents 查看可用资源"}

    except SystemError as e:
        logger.critical(f"系统错误: {e}", exc_info=True)
        alert_admin(e)
        return {"success": False,
                "error": {"error_code": "INTERNAL_ERROR",
                         "message": "系统内部错误，请联系管理员"}}

模式 2：异常链（Exception Chaining）

核心思想：保留原始异常信息，方便调试

❌ 丢失原始异常
except OSError:
    raise FileSystemError(
        f"无法写入文件 {path}"
    )
✅ 保留异常链
except OSError as e:
    raise FileSystemError(
        operation="write",
        path=path,
        reason=str(e)
    ) from e

模式 3：重试装饰器（Retry Decorator）

核心思想：自动重试可恢复错误

适用场景：调用外部服务（LLM API、网络请求）

def retry_on_recoverable(
    max_attempts: int = 3,
    backoff_factor: float = 2.0,
    max_backoff: float = 60.0
):
    """重试装饰器，只重试 RecoverableError"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            attempt = 0
            while attempt < max_attempts:
                try:
                    return await func(*args, **kwargs)
                except RecoverableError as e:
                    attempt += 1
                    if attempt >= max_attempts:
                        raise
                    delay = min(
                        e.retry_after * (backoff_factor ** (attempt - 1)),
                        max_backoff
                    )
                    await asyncio.sleep(delay)
                except Exception:
                    raise
        return wrapper
    return decorator

# 使用示例
@retry_on_recoverable(max_attempts=3)
async def call_llm_api(prompt):
    response = await llm_client.generate(prompt)
    if response.status == 429:
        raise RateLimitError("API 限流", retry_after=5.0)
    return response.text

模式 4：上下文管理器（Context Manager）

核心思想：自动管理资源和状态，即使发生异常

from contextlib import asynccontextmanager

@asynccontextmanager
async def agent_execution_context(agent_name, call_id):
    """Agent 执行上下文，自动管理状态和清理"""
    # 1. Setup：进入时执行
    call_manager.update_status(call_id, CallStatus.RUNNING)

    try:
        yield  # 2. 暂停，执行业务逻辑
        # 3a. Teardown（成功）
        call_manager.update_status(call_id, CallStatus.COMPLETED)
    except Exception as e:
        # 3b. Teardown（失败）
        call_manager.update_status(call_id, CallStatus.FAILED)
        raise
    finally:
        # 3c. 总是执行
        logger.debug(f"清理 Agent {agent_name} 的资源")

# 使用示例
async def execute_agent(agent_name, call_id, prompt):
    async with agent_execution_context(agent_name, call_id):
        result = await agent.execute(prompt)
        return result

模式 5：错误恢复策略

核心思想：根据错误类型选择不同的恢复策略

async def process_message_with_recovery(agent, message):
    try:
        result = await agent.execute(message.content)
        return result

    except RecoverableError as e:
        raise  # 由装饰器处理重试

    except ValidationError as e:
        await send_error_to_sender(message.send_from, e)

    except AgentExecutionError as e:

    except SystemError as e:
        logger.critical(f"系统错误: {e}", exc_info=True)
        alert_admin(e)
        raise

日志级别使用指南

级别使用场景示例

结构化日志示例

class StructuredLogger:
    """结构化日志记录器"""

    def __init__(self, name: str):
        self.logger = logging.getLogger(name)

    def log_error(self, error, context=None, include_traceback=True):
        """记录错误，包含上下文信息"""
        log_data = {
            "timestamp": datetime.now().isoformat(),
            "error_type": type(error).__name__,
            "error_message": str(error),
            "context": context or {}
        }
        if isinstance(error, AgentsHubError):
            log_data["error_code"] = error.error_code
            log_data["details"] = error.details

        self.logger.error(
            json.dumps(log_data, ensure_ascii=False),
            exc_info=include_traceback
        )

错误处理反模式

需要避免的常见错误
❌ 吞掉异常
try:
    result = do_something()
except Exception:
    pass  # 完全忽略错误
✅ 正确处理
try:
    result = do_something()
except Exception as e:
    logger.warning(f"操作失败: {e}")
    result = default_value
❌ 捕获过于宽泛
try:
    result = do_something()
except Exception:  # 捕获所有
    handle_error()
✅ 精确捕获
try:
    result = do_something()
except (ValueError, TypeError) as e:
    handle_error(e)
❌ 异常作为控制流
try:
    user = users[user_id]
except KeyError:
    user = create_user(user_id)
✅ 用返回值判断
user = users.get(user_id)
if user is None:
    user = create_user(user_id)
❌ 丢失异常链
except RequestException:
    raise CustomError("API 调用失败")
✅ 保留异常链
except RequestException as e:
    raise CustomError("API 调用失败") from e
❌ 循环中吞掉异常
for item in items:
    try:
        process(item)
    except Exception:
        continue  # 跳过所有错误
✅ 精确跳过
for item in items:
    try:
        process(item)
    except ProcessError as e:
        logger.warning(f"处理 {item} 失败: {e}")
        continue

案例 1：文件操作错误处理

async def save_group_chat_session(self):
    """保存群聊会话，完善的错误处理"""
    try:
        os.makedirs(self.group_chat_session_path, exist_ok=True)
        self.group_chat_session.updated_at = datetime.now()

        with open(self.messages_file, 'w', encoding='utf-8') as f:
            meta_data = {
                '_type': 'meta_data',
                'updated_at': self.group_chat_session.updated_at.isoformat()
            }
            f.write(json.dumps(meta_data, ensure_ascii=False) + '\n')
            for msg in self.group_chat_session.messages:
                f.write(json.dumps(msg, ensure_ascii=False) + '\n')

    except PermissionError as e:
        raise FileSystemError(
            operation="write",
            path=self.messages_file,
            reason="权限不足"
        ) from e

    except OSError as e:
        if e.errno == 28:  # No space left on device
            raise FileSystemError(
                operation="write",
                path=self.messages_file,
                reason="磁盘空间不足"
            ) from e
        raise FileSystemError(
            operation="write",
            path=self.messages_file,
            reason=str(e)
        ) from e

async def compact_messages(self, agent_info):
    """压缩群聊消息历史，带重试和降级"""
    uncompacted = self.group_chat_session.get_uncompact_messages()
    if not uncompacted:
        return

    token_count = estimate_prompt_tokens(uncompacted)
    if token_count < MAX_TOKEN:
        return

    compact_prompt = build_compact_prompt(uncompacted, agent_info)

    for attempt in range(3):
        try:
            compact_result = await llm_call.execute(compact_prompt)
            compact_data = parse_compact_result(compact_result.text)
            await save_compact_record(compact_data)
            self.group_chat_session.last_compacted_loc = len(
                self.group_chat_session.messages
            )
            self.save_group_chat_session()
            return

        except json.JSONDecodeError as e:
            logger.warning(f"第 {attempt+1} 次压缩失败: {e}")
            if attempt < 2:
                await asyncio.sleep(2 ** attempt)
                continue
            await self._fallback_compact(uncompacted)
            return

        except LLMAPIError as e:
            if attempt < 2:
                await asyncio.sleep(2 ** attempt)
                continue
            raise CompactionError(f"压缩失败: {str(e)}") from e
