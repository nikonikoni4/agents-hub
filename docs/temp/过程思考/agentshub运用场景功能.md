# 调研-学习-深入思考
当前有3个skill
1. knowledge-research ： 进行调研产出知识图谱
2. knowledge-learning : 学习具体的内容
3. deep-answer:针对于某个问题进行具体的回答

场景是：
1. 告诉leader想要学习什么内容，告诉当前的学习状态，决定调研的详细程度
2. leader让research准备知识图谱（教材）
3. 调研完毕之后，开始learning
4. learning过程中，如果发现教程不完整，需要补充某些内容，可以让research再次进行调研和补充
5. 对于深度的问题，由deep-answer来回答，learning就想像是普通教师，只是告诉是什么，而deep-answer，获取信息能力更强，不局限于知识图谱，而且提供批判性思维角度，包含方案选择等有关为什么的内容

这里补充的短板是，1. 3个角色都是必须要隔离上下文的 2. 解决了直接在终端无法进行A2A的问题

## 需要的功能
1. A2A方面可以由TL进行统一调度，或者可以直接learning想research提出要求，需要补充什么材料。两种方案都行
2. 可以共享用户的学习状态
