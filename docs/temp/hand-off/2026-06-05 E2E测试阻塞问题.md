## 二、阻塞问题

### 核心阻塞：创建群聊 API 返回 500 错误

**API**：`POST /api/v1/group-chats`

**错误响应**：
```json
{"error_code":"INTERNAL_ERROR","message":"服务器内部错误","type":"InternalError"}
```

**调用链**：
```
POST /api/v1/group-chats
  → GroupChatService.create_group_chat()
    → group_chat.start()
      → _initialize_new_members()
        → agent.execute()  ← 失败点
          → ClaudeExecutor.execute()  ← 调用 Claude CLI
```

**根本原因**：
`group_chat.start()` 在 `_initialize_new_members()` 中会为每个新成员调用 Claude CLI 执行初始化对话（打招呼）。这个过程失败了，但具体错误信息被日志截断。

**日志截断问题**：
- 日志文件：`local_data/logs/agents_hub.log`
- 截断显示：`[截断, 原长 3454]`
- `MAX_LOG_MESSAGE_LENGTH` 已设置为 4000，但仍然在 3454 处截断
- 可能原因：使用了其他分支的包（用户已重新 `pip install -e .`，但问题未解决）
