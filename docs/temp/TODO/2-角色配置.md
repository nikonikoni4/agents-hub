# 角色配置

2. agents设置role的相关配置，主要作用是
    初始化：
        1. 动态注册role， 2. 每个role会在LOCAL_DATA_PATH = "local_data" 下创建一个agents以role为名称的文件夹 具体为 `LOCAL_DATA_PATH / agents / <role_name>` 3. 核心功能是配置claude code / codex 的全局目录文件 4. 默认claude 平台的角色复制 `Path.home() / .claude `的settings.json ，创建空白的CLAUDE.md 5. codex 从`Path.home() / .codex ` 复制 auth.json 、 config.toml和 rules文件夹 
        6. 配置角色metadata role.json：
            ```json
            {
                "name": "角色名称",
                "type": "leader|team_member",
                "scope": [""], # 所属群聊
                "avatar": "/path/to/avatar.png",
                "abilities": ""
            }
            ```
    修改配置
        1. 增加skill，迁移新的skill到 `local_data / agents / <role> / work_root/  skills`
        2. 修改settings.json 或 config.toml 
        3. 修改settings.json 或 config.toml 的部分内容的快捷方式，主要是为前端快速设置提供方法
            1. 对于claude code：修改permission模式，修改allow,deny ask
            2. 对于codex
        4. 选择头像，上传头像
        5. 返回结构化数据： role_name,ability,prompt
