# 出现的问题

## 1. 环境配置问题

### 1.1 Mock 模式未关闭
- **问题**：`.env.development` 中 `VITE_USE_MOCK=true`，导致前端使用 mock 数据
- **解决**：改为 `VITE_USE_MOCK=false`

### 1.2 CORS 跨域问题
- **问题**：前端直接请求 `http://localhost:8099/api/v1/...`，存在跨域限制
- **原因**：`.env.development` 中 `VITE_API_BASE_URL=http://localhost:8099/api/v1`
- **解决**：改为 `VITE_API_BASE_URL=/api/v1`，通过 Vite proxy 代理

### 1.3 Vite Proxy 端口不匹配
- **问题**：`vite.config.ts` 中 proxy 目标端口为 `8000`，但后端运行在 `8099`
- **解决**：修改 `target: 'http://localhost:8099'`

---

## 2. 后端 API 问题

### 2.1 创建群聊返回 500 错误
- **问题**：`POST /api/v1/group-chats` 返回 `500 INTERNAL_ERROR`
- **错误信息**：`群聊启动失败: [ROLE_NOT_FOUND] Role 'manager' 不存在`
- **原因**：后端创建群聊时需要一个名为 `manager` 的角色
- **临时解决**：通过 API 预先创建 `manager` 角色
- **后续问题**：创建 `manager` 角色后，仍然返回 500 错误，可能是 `group_chat.start()` 内部逻辑问题

### 2.2 manager 角色问题
- manager 应该是默认创建的系统角色，前端不能添加
- 在创建群聊时应该是默认选择且不可取消 manager
- 当前使用的是 Leader 应该改为 manager
- 应该在程序初始化的时候增加 manager（默认创建）：1. 先判断是否存在 2. 不存在则创建

### 2.3 错误处理问题
- 当前错误处理可能还存在把错误传递给吞了的问题（仅 raise 但是没有 `from e`）

### 2.4 部分 API 返回 404
- `GET /api/v1/group-chats/{id}/members` 对某些群聊 ID 返回 404

## 4. Playwright 测试技巧

### 4.1 使用 label 点击隐藏的 radio/checkbox
```python
# 正确方式：点击 label
page.locator('label:has-text("E2E测试Leader")').click()

# 或者使用 JavaScript
page.evaluate("""() => {
    const radio = document.querySelector('input[type="radio"][name="leader"]');
    if (radio) {
        radio.click();
        radio.dispatchEvent(new Event('change', { bubbles: true }));
    }
}""")
```

### 4.2 选择器精确度
- `get_by_text("角色管理")` 匹配到多个元素
- 使用 `get_by_role("button", name="角色管理").nth(1)` 精确定位

### 4.3 网络请求监听
```python
def handle_response(response):
    if "/api/v1/" in response.url and not response.url.endswith(".ts"):
        print(f"[API] {response.status} {response.url}")

page.on("response", handle_response)
```

# 已解决的内容

1. Mock 模式切换：`VITE_USE_MOCK=false`
2. CORS 跨域问题：使用 Vite proxy
3. Vite proxy 端口配置：修改为 8099
4. 隐藏表单元素点击：使用 JavaScript evaluate
5. 选择器精确度：使用 nth() 索引
